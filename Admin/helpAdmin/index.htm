СПРАВКА АДМИНИСТРАТОРА

Главное окно

При входе в программный комплекс администратора будет ожидать главное окно с выбором базы данных/библиотеки и таблицы.

          После выбора интересующей базы данных/библиотеки необходимо нажать на кнопку «Открыть таблицу» в нижней части окна. После этого откроется редактор этой таблицы.

 

Редактор таблиц

Все редакторы представляют собой окно с кнопками добавления записи (), удаления выделенных записей (), а также сохранения изменений (). Ниже панели с этими элементами управления расположена таблица с записями. Исключениями являются редакторы таблиц функций Matlab, математических моделей и методов нелинейного программирования. Они содержат некоторые дополнительные элементы управления.

          Открытая выше таблица не имеет внешних зависимостей, лишь один столбец наименования, поэтому в таблице нет столбца с выпадающим списком. Редактор таблицы, имеющей внешние зависимости, например «Режимные параметры процесса» будет иметь выпадающие списки с выбором записи из другой таблицы.

Здесь внешними зависимостями являются «Установка риформинга», «Параметр», «Математическая модель», «Единица измерения». При добавлении новой записи и раскрытии списка одного из таких столбцов программа может отобразить, что какое-либо значение уже подставлено, однако, его всё равно необходимо выбрать вручную.

Программа поддерживает множественное удаление записей. Для этого необходимо зажав кнопку «Shift» нажать на заголовок строки, по который необходимо произвести удаление , после чего нажав на кнопку , подтвердить удаление. То же можно проделать только с зажатой кнопкой «Ctrl», тогда записи будут выбираться по одной, а не диапазоном.

 

Пример добавления математической модели

         В качестве примера добавим модель из [1]. Для этого нам необходимо написать две функции Matlab – для расчёта отбора и октанового числа риформинг-бензина. Зайдём в редактор Matlab, создадим файл, и напишем следующий код:

function [F] = BeltsovLinearSelection(In,Ec)

T = In('T');

G = In('G');

yN = In('yN') / 100;

yA = In('yA') / 100;

a0 = Ec('a0');

a1 = Ec('a1');

a2 = Ec('a2');

a3 = Ec('a3');

F=a0+a1*T+a2*G+a3*(yN+yA);

end

Здесь function [F] = BeltsovNonLinearSelection(In,Ec) – сигнатура функции. В ней

[F] – вычисленное значение функции (выходной аргумент), BeltsovLinearSelection – имя функции, (In,Ec) – входные аргументы, где In – переменная типа «словарь», где ключами выступают обозначения параметров в виде строковых литералов, а значениями – значения этих параметров. Это могут быть как входные параметры (состав сырья), так и режимные (температура сырья на входе в реакторы, подача сырья). Такой же тип данных имеет и переменная Ec, представляющая собой уже набор эмпирических коэффициентов. В строках

T = In('T');

G = In('G');

yN = In('yN') / 100;

yA = In('yA') / 100;

происходит получение значений параметров из переменной In. Строки

a0 = Ec('a0');

a1 = Ec('a1');

a2 = Ec('a2');

a3 = Ec('a3');

обозначают получение значений эмпирических коэффициентов из Ec.

В строке F=a0+a1*T+a2*G+a3*(yN+yA); рассчитывается значение отбора риформинг бензина с использованием полученных значений параметров и эмпирических коэффициентов.

Сохраним созданный файл с названием, совпадающим с наименованием функции и расширением «.m» в удобное место с помощью сочетания клавиш Ctrl+S.

          Создадим ещё один файл, который будет содержать код для расчёта октанового числа:

function [N] = BeltsovLinearOctane(In, Ec)

T = In('T');

G = In('G');

yN = In('yN') / 100;

yA = In('yA') / 100;

b0 = Ec('b0');

b1 = Ec('b1');

b2 = Ec('b2');

b3 = Ec('b3');

N=b0+b1*T+b2*G+b3*(yN+yA);

end

          Функция предельно похожа на предыдущую за исключением обозначений эмпирических коэффициентов. Сохраним и этот файл.

          В таблице «Единицы измерения» определим «т/ч» для функции отбора риформинг-бензина.

          Добавим сохранённые функции в наш комплекс. Для этого в главном окне администратора выберем «Библиотеку математических моделей» и «Функции Matlab», откроем таблицу.

Нажмём на , введём наименование новой функции, например, «Отбор лин.», и нажмём кнопку «Импортировать функцию Matlab». Далее в окне выбора файла найдём раннее сохранённую функцию отбора и нажмём «Ок». Укажем единицу измерения. То же сделаем и для функции октанового числа с названием «Октановое число лин.» без выбора единицы измерения, так как это безразмерная величина. Сохраним изменения () и закроем таблицу. Примечание – программный комплекс только проверяет сигнатуру функции. В ней должно содержаться: 1 выходной параметр – полученное значение функции; наименование функции; в круглых скобках 2 входных аргумента: словарь обозначений и значений параметров; словарь обозначений и значений эмпирических коэффициентов. Комплекс не отслеживает корректность кода Matlab внутри функции, а также адекватность значений, получаемых из этой функции, это лежит на плечах администратора. Кроме того, после добавления функции в комплекс и сохранения изменений, файл с функцией с компьютера можно удалить, файл сохраняется комплексом в свою базу данных. Скриншот ниже демонстрирует ещё две функции, с нелинейными моделями, они были добавлены заранее.

          Создадим математическую модель, содержащую эти две функции. Для этого выберем таблицу «Математические модели».

          Добавим строку. Назовём модель «Линейная модель». В выпадающих списках столбцов «Функция отбора риформинг-бензина» и «Функция октанового числа риформинг-бензина» выберем созданные нами функции «Отбор лин.» и «Октановое число лин.» соответственно.

          Укажем источник математической модели в окне, открываемом при нажатии кнопки «Изменить описание» и нажмём кнопку «ОК».

Сохраним изменения и закроем таблицу.

          Последним шагом станет определение эмпирических коэффициентов модели в таблице «Эмпирические коэффициенты».

 

Пример добавления характеристик процесса и связывания с математической моделью

          Создадим запись об установке риформинга в «Базе данных характеристик процесса» и «Установках риформинга».

          Свяжем созданную запись об установке с созданной в предыдущем примере записью математической модели в таблице «Математические модели установок риформинга».

Создадим записи о сырье, и катализаторе в таблицах «Сырьё» и «Катализаторы» соответственно.

          Создадим записи о реакторах в таблице «Реакторы» и укажем созданную запись установки в выпадающих списках.

          Свяжем созданные записи о сырье и катализаторе с записью установки, чтобы обозначить, что в эту установку можно загрузить это сырьё и катализатор. Сделать это можно в таблицах «Сырьё установок риформинга» и «Катализаторы установок риформинга».

          Добавим в таблицу «Единицы измерения» единицы измерения в будущем определяемых параметров.

          Далее в «Справочнике параметров» определим наименования будущих параметров.

          В таблице «Характеристики сырья» определим значения и единицы измерения параметров «Содержание нафтеновых углеводородов», «Содержание ароматических углеводородов», «Молярный вес сырья», «Плотность сырья» для двух созданных записей сырья «Фракции прямогонного бензина 100-165 °C» и «Фракции прямогонного бензина 70-140 °C», а также созданной в предыдущем пункте записи линейной математической модели.

          Стоит упомянуть, что некоторые параметры тут излишни для этой математической модели («Молярный вес сырья и «Плотность сырья») и служат для лучшего информирования исследователем характеристик сырья. Однако при расчётах они всё равно будут переданы в функции отбора и октанового числа модели.

          Сделаем то же для «Характеристик реакторов» и «Характеристик катализаторов».

          Далее определим режимные параметры процесса в соответствующей таблице.

         

Пример добавления метода нелинейного программирования и связывания с математической моделью

          Снова создадим файл Matlab и напишем следующий код:

function [x, fval, flag, iterations, funcCount, firstorderopt, elapsedTime, Er] = ActiveSet(VPLBs, VPUBs, P0, OMP, TF, FoF, OutFun)

if (isKey(OMP,'t'))

    t=timer('TimerFcn', @(x,y)StopFunc, 'StartDelay', OMP('t'));

    start(t);

end

tic

timeIsOverLimit=false;

 

options=optimset('outputfcn', @outfun);

 

if(isKey(OMP,'mFE'))

    options=optimset(options,'MaxFunEvals', OMP('mFE'));

end

 

if(isKey(OMP,'mI'))

    options=optimset(options, 'MaxIter', OMP('mI'));

end

 

if(isKey(OMP,'tF'))

    options=optimset(options, 'TolFun', OMP('tF'));

end

options = optimset(options, 'Algorithm', 'active-set');

 

[x,fval,flag,output] = fmincon(TF, P0, [], [], [], [], VPLBs, VPUBs, FoF, options);

 

iterations=output.iterations;

funcCount=output.funcCount;

firstorderopt=output.firstorderopt;

elapsedTime=toc;

if (flag==-2)

    Er=['Не удалось получить решение. Это может быть вызвано слишком большим значением критериального ограничения, ' ...

        'попробуйте его снизить.'];

else

    Er='';

end

if(isKey(OMP,'t'))

    stop(t);

    delete(t);

end

 

    function [stop] = outfun(x, optimValues, state)

    stop=timeIsOverLimit;

    OutFun(x, optimValues, state);

    end

 

    function StopFunc()

    timeIsOverLimit = true;

    end

end

[x, fval, flag, iterations, funcCount, firstorderopt, elapsedTime, Er] – выходные аргументы, где x – точка, при которой достигается максимум целевой функции, fval – максимальное значение целевой функции, flag – флаг, получаемый по окончании работы метода нелинейного программирования Matlab, iterations – количество итераций метода, funcCount – количество подсчёта целевой функции, firstorderopt – мера оптимальности первого порядка, elapsedTime – время работы метода, Er – сообщение об ошибке. ActiveSet – название функции. VPLBs – массив нижних границ варьируемых параметров, VPUBs – массив верхних границ варьируемых параметров, P0 – стартовая точка, OMP – словарь с обозначениями и значениями параметров метода, TF – целевая функция, FoF – функция критериального ограничения (с уже встроенным значением), OutFun – функция записи точек, сгенерированных методом нелинейного программирования.

Строка tic запускает отсчёт времени работы метода, а строки

if (isKey(OMP,'t'))

    t=timer('TimerFcn', @(x,y)StopFunc, 'StartDelay', OMP('t'));

    start(t);

end

timeIsOverLimit=false;

 

проверяют, есть ли в словаре параметров метода параметр с обозначением «t», означающим ограничение по времени работы. Если да, то создаётся таймер, в который передаётся функция, которая вызывается по истечению времени («StopFunc»), а также само значение ограничения по времени («’StartDelay’, OMP(‘t’)), таймер запускается, создаётся переменная «timeIsOverLimit», которой в функции «StopFunc» будет присвоено значение «true», означающее то, что таймер достиг заданного времени работы. Функция «StopFunc»:

function StopFunc()

timeIsOverLimit = true;

end

В строке options=optimset('outputfcn', @outfun); создаётся набор настроек для метода, в который сразу помещается функция, вызываемая методом на каждой итерации. Вот эта функция:

function [stop] = outfun(x, optimValues, state)

stop=timeIsOverLimit;

OutFun(x, optimValues, state);

end

Она возвращает флаг прекращения работы метода, принимает в себя текущую точку x, значение целевой функции optimValues, состояние метода на итерации state. В строке stop=timeIsOverLimit; флагу прекращения работы метода присваивается флаг достижения таймера времени работы метода. Далее вызывается переданная во входных аргументах функция «OutFun», которая фиксирует сгенерированную на итерации метода точку.

Строки

if(isKey(OMP,'mFE'))

    options=optimset(options,'MaxFunEvals', OMP('mFE'));

end

 

if(isKey(OMP,'mI'))

    options=optimset(options, 'MaxIter', OMP('mI'));

end

 

if(isKey(OMP,'tF'))

    options=optimset(options, 'TolFun', OMP('tF'));

end

ищут в словаре параметров метода значения этих параметров и обновляют набор настроек.

      Строка options = optimset(options, 'Algorithm', 'active-set'); обновляет набор настроек, выбирая метод активного набора в качестве метода нелинейного программирования.

          Строка [x,fval,flag,output] = fmincon(TF, P0, [], [], [], [], VPLBs, VPUBs, FoF, options); запускает процесс оптимизации.

          Строки

iterations=output.iterations;

funcCount=output.funcCount;

firstorderopt=output.firstorderopt;

получают количество итераций метода, число подсчёта целевой функции, меру оптимальности первого порядка.

          Строка elapsedTime=toc; - останавливает отсчёт работы метода и записывает время работы метода в соответствующий выходной аргумент. Строки

if (flag==-2)

    Er=['Не удалось получить решение. Это может быть вызвано слишком большим значением критериального ограничения, ' ...

        'попробуйте его снизить.'];

else

    Er='';

end

проверяют флаг прекращения работы метода и задают значение выходного аргумента сообщения об ошибке.

Строки

if(isKey(OMP,'t'))

    stop(t);

    delete(t);

end

останавливают и удаляют таймер.

          Замечания тут такие же, как и при добавлении функции отбора/октанового числа. Изменилась только сигнатура функции.

          Сохраним этот файл под названием, соответствующим названию функции и расширением «.m».

          В главном окне выберем «Библиотеку методов нелинейного программирования», «Методы нелинейного программирования Matlab». Создадим новую запись, назовём метод «Метод активного набора», нажмём кнопку «Импортировать метод оптимизации Matlab», выберем сохранённый нами файл, нажмём «ОК». При желании можно указать описание метода. Сохраним изменения и закроем таблицу.

          Свяжем этот метод с добавленными раннее функциями отбора и октанового числа. Для этого перейдём в таблицу «Поддерживаемые методами нелинейного программирования функции Matlab», укажем добавленный метод и функции.

 

Так как в коде мы указали параметры метода, необходимо также указать их и в библиотеке методов нелинейного программирования. Для этого перейдём в «Справочник методов нелинейного программирования» и создадим указанные в коде метода параметры.

          Далее перейдём в «Параметры методов нелинейного программирования», свяжем их с добавленным методом, укажем границы, единицы измерения и другие данные.
